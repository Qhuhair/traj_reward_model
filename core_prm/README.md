# Core-PRM — 过程奖励计算引擎

## 整体目的

LLM 对每一步打出的 Q 值是**单步视角**的孤立评分。core_prm 用**时序数学模型**对 Q 序列进行二次加工，产出三个指标：

| 指标 | 回答的问题 |
|------|----------|
| Progress (Δ) | 这一步相对于历史趋势产生了多少"价值增量"？ |
| TD-Error (δ) | 这一步的实际贡献与预期有多大偏差？ |
| GAE (A^GAE) | 综合考虑全轨迹后，这一步的**全局真实优势**是多少？ |

## 输入与输出

**输入**: LLM 产出的 Q 值序列
```
[Q_0, Q_1, Q_2, ..., Q_T]   (每个 Q ∈ [0.0, 1.0])
```

**输出**: `EstimationResult`
```json
{
  "progress":   [Δ_0,  Δ_1,  ..., Δ_T],
  "td_errors":  [δ_0,  δ_1,  ..., δ_T],
  "advantages": [A_0,  A_1,  ..., A_T],
  "method": "TDGAEEngine(γ=0.99, λ=0.95)"
}
```

## 算法详解

以下全部以 `traj_002` 最近一次运行的 Q 值序列为例：
```
Q = [1.000, 0.100, 1.000, 0.950, 0.750, 0.950, 0.200, 1.000]
```
（8 步：首页→酒店搜索→错误点会员→修正→选酒店→点Back→连续Back→重新点会员）

---

### 算法一：Progress — 简单进展

**公式**:

$$A_t = Q_t - V_t$$

其中 $V_t$ 是基准值 (baseline)，由配置 `progress.baseline_method` 决定：

| baseline_method | $V_t$ 定义 |
|----------------|-----------|
| `running_mean` | $V_t = \frac{1}{t}\sum_{i=0}^{t-1} Q_i$ (历史均值, t>0 时) |
| `prev_q` | $V_t = Q_{t-1}$ (上一时刻) |
| `zero` | $V_t = 0$ |

**直觉**: 如果某一步的 Q 值**显著高于之前所有步的平均值**，说明这一步是"实质性推进"。如果 Q 值低于历史平均，说明在"原地打转"。

**traj_002 实例 (running_mean)**:

```
Step  Q_t     V_t (历史均值)            Δ_t         解读
  1   1.000   0.000 (初始为0)          +1.000       远超基准 — 重大推进
  2   0.100   1.000 (=1.000/1)         -0.900       暴跌 — 偏离主线
  3   1.000   0.550 (=(1+0.1)/2)       +0.450       回到正确方向
  4   0.950   0.700 (=(1+0.1+1)/3)     +0.250       继续推进
  5   0.750   0.763 (=(2.1+0.95)/4)    -0.013       略低于均值 — 轻微停滞
  6   0.950   0.760 (=(3.05+0.75)/5)   +0.190       恢复推进
  7   0.200   0.792 (=(3.8+0.95)/6)    -0.592       显著倒退
  8   1.000   0.707 (=(4.75+0.2)/7)    +0.293       跳回正确终点
```

---

### 算法二：TD-Error — 时间差分

**公式**:

$$\delta_t = r_t + \gamma \cdot Q_t - Q_{t-1}$$

其中:
- $\gamma = 0.99$ (折扣因子, `gamma`)
- $r_t$ = `terminal_reward` (1.0, 仅最后一步) 或 `non_terminal_reward` (0.0)
- $Q_{-1} = 0.0$

**直觉**: TD-Error 从数学上比较"当前步的价值 + 未来折现"与"上一步价值"的差额。$\delta > 0$ 说明当前步的决策**超越了上一步的价值预期**；$\delta < 0$ 说明**不如直接卡在上一步**。

**traj_002 实例**:

```
Step  Q_t     Q_{t-1}   r_t    δ_t = r_t + 0.99·Q_t - Q_{t-1}
  1   1.000   0.000     0.0    0.0 + 0.990 - 0.000  = +0.990   ← 从0到1的跃迁
  2   0.100   1.000     0.0    0.0 + 0.099 - 1.000  = -0.901   ← 暴跌: 做错了
  3   1.000   0.100     0.0    0.0 + 0.990 - 0.100  = +0.890   ← 强力修正
  4   0.950   1.000     0.0    0.0 + 0.941 - 1.000  = -0.060   ← 微跌
  5   0.750   0.950     0.0    0.0 + 0.743 - 0.950  = -0.208   ← 倒退
  6   0.950   0.750     0.0    0.0 + 0.941 - 0.750  = +0.191   ← 恢复
  7   0.200   0.950     0.0    0.0 + 0.198 - 0.950  = -0.752   ← 严重倒退
  8   1.000   0.200     1.0    1.0 + 0.990 - 0.200  = +1.790   ← 终端奖励+满分
```

**关键解读**:
- Step 2: δ=-0.901 — 从 Q=1.0 暴跌到 0.1，模型认为点击会员权益是个**严重偏离** (确实，子目标是"去酒店搜索"而非"看会员权益")
- Step 3: δ=+0.890 — 点击搜索栏后强力修正，回到了正确的北京酒店搜索结果页
- Step 8: δ=+1.790 — 包含 terminal_reward=1.0，表示**最终成功到达目标**
- γ=0.99 意味着"未来价值几乎不打折"，适合短轨迹 (8-10 步)

---

### 算法三：GAE — 广义优势估计

**公式**:

$$A_t^{GAE} = \sum_{k=0}^{\infty} (\gamma \lambda)^k \cdot \delta_{t+k}$$

**递推实现** (从后向前):

$$A_T = \delta_T$$

$$A_t = \delta_t + \gamma \lambda \cdot A_{t+1}$$

**直觉**: TD-Error 只看相邻两步，GAE 把**未来所有步的 TD-Error** 按指数衰减 $(\gamma\lambda)^k$ 加权求和。λ=0.95 时：
- 当前步 δ 权重 = 1
- 后 1 步 δ 权重 = 0.94 (=0.99×0.95)
- 后 2 步 δ 权重 = 0.88
- 后 3 步 δ 权重 = 0.83
- ...逐步衰减

这样 Step 1 的 GAE 不仅反映第一步做得对，还**汇聚了整条轨迹最终成功的红利**。

**traj_002 GAE 计算过程** ($\gamma\lambda = 0.9405$):

```
Step  δ_t      A_t^GAE (从后往前递推)
  8   +1.790   A_8 = 1.790                                                   = +1.790
  7   -0.752   A_7 = -0.752 + 0.9405·1.790 = -0.752 + 1.683      = +0.931
  6   +0.191   A_6 = +0.191 + 0.9405·0.931 = +0.191 + 0.876      = +1.067
  5   -0.208   A_5 = -0.208 + 0.9405·1.067 = -0.208 + 1.004      = +0.796
  4   -0.060   A_4 = -0.060 + 0.9405·0.796 = -0.060 + 0.749      = +0.689
  3   +0.890   A_3 = +0.890 + 0.9405·0.689 = +0.890 + 0.648      = +1.538
  2   -0.901   A_2 = -0.901 + 0.9405·1.538 = -0.901 + 1.447      = +0.545
  1   +0.990   A_1 = +0.990 + 0.9405·0.545 = +0.990 + 0.513      = +1.503
```

**核心洞察**: Step 2 的 TD-Error 是 -0.901（单独看很糟糕），但 GAE 是 +0.545（可以接受）。原因：虽然第 2 步点错了，但**后续轨迹修正过来了** (Step 3 强力扭转)，最终到达目标。GAE 平滑了单步噪声，给出更全局的评价。

## 三维对比与训练数据决策

| Step | Q (单步评分) | Progress (Δ) | TD-Error (δ) | GAE (全局) | 结论 |
|------|-------------|-------------|-------------|-----------|------|
| 1 | 1.000 | +1.000 ⬆ | +0.990 ⬆ | +1.503 ⬆ | ✅ 正确起步 |
| 2 | 0.100 | -0.900 ⬇ | -0.901 ⬇ | +0.545 → | ⚠️ 单步差, 全局可接受 |
| 3 | 1.000 | +0.450 ⬆ | +0.890 ⬆ | +1.538 ⬆ | ✅ 强力修正 |
| 4 | 0.950 | +0.250 ⬆ | -0.060 → | +0.689 → | ✅ 继续推进 |
| 5 | 0.750 | -0.013 → | -0.208 ⬇ | +0.796 ⬆ | ⚠️ 轻微停滞 |
| 6 | 0.950 | +0.190 ⬆ | +0.191 ⬆ | +1.067 ⬆ | ✅ 恢复 |
| 7 | 0.200 | -0.592 ⬇ | -0.752 ⬇ | +0.931 ⬆ | ⚠️ 短期倒退但没破坏全局 |
| 8 | 1.000 | +0.293 ⬆ | +1.790 ⬆ | +1.790 ⬆ | ✅ 终点巨大红利 |

**训练数据用途**: GAE 高的步骤 → 高权重训练样本；Progress 持续为负的步骤 → 标记可疑，可能剔除或降权。

## 设计模式

| 模式 | 实现 |
|------|------|
| **适配器模式** | `LLMPromptAdapter` 统一处理 LLM 文本 / JSON steps / 直接 Q 序列三种输入 → 标准化 `AdaptedData` |
| **策略模式** | `BaseEstimator` → `ProgressCalculator` (简单差分) / `TDGAEEngine` (复杂优势估计)，config 切换 |
| **单一职责** | adapter 管解析, estimator 管计算, orchestrator 管调度 |

## 扩展指南

| 需求 | 如何实现 |
|------|---------|
| 新增数据源 | `data_adapters/` 下新增 Adapter，继承 `BaseAdapter` |
| 新增估计算法 | `estimators/` 下新增类，继承 `BaseEstimator`，在 config 注册 |
| 调整 γ/λ | 修改 `config/prm_params.yaml` 即可 |

## 超参数配置

`config/prm_params.yaml`:
```yaml
gamma: 0.99              # 折扣因子 — 近 1.0 适合短轨迹
lambda_gae: 0.95         # GAE 衰减 — 近 1.0 更平滑但引入远期噪声
estimator: "td_gae"      # 选 "td_gae" 或 "progress"
progress:
  baseline_method: "running_mean"
td_gae:
  terminal_reward: 1.0
  non_terminal_reward: 0.0
```
